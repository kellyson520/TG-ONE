# Worker 性能提升任务报告 (Worker Performance Boost Report)

## 1. 任务概述
提升 Worker 处理速度并降低资源占用，针对 VPS 高负载（300% CPU）情况进行架构层面的优化。

## 2. 优化指标对比
| 指标 | 优化前 (预期) | 优化后 (目标) | 状态 |
| :--- | :--- | :--- | :--- |
| CPU 最高占用 | 300%+ | < 150% | ⌛ |
| 内存峰值 | 1.8GB+ | < 1.2GB | ⌛ |
| DB 锁定错误速率 | 高频率 | 偶发/无 | ⌛ |
| 任务处理吞吐量 | ~20 tps | > 50 tps | ⌛ |

## 3. 核心变更方案
1. **Producer-Consumer 模式**: 消除 N 个 Worker 同步抢占 DB 的锁竞争。
2. **批量拉取**: 提高 IO 效率。
3. **主动内存管理**: 防止长期运行的内存溢出。
4. **精细化资源监控**: 引入 Loop Lag 监控防死锁。
