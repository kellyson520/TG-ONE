在不依赖任何外部大模型（LLM）或厚重的预训练NLP模型（如Word2Vec、BERT）的情况下，**“自学习”的本质是建立动态的“统计学反馈闭环”和“行为特征画像”**。

你要让算法自己“长记性”，即：**通过历史数据不断修正当下的判决阈值。** 以下是纯粹基于统计学、数据结构和启发式规则，提升自学习算法准确度和可信度的 4 个核心策略，这些都可以直接在你的 `services/hotword_learn.py` 或相关逻辑中落地：

### 1. 动态基线与变异系数学习（惩罚“常态词”）

**自学习目标：** 让系统自动识别并屏蔽那些“永远都在霸榜”的结构性词汇（如“群组”、“频道”）。
**算法实现：变异系数 (Coefficient of Variation, CV)**

* **记录历史：** 在数据库或 Redis 中维护每个词过去 7 天或 14 天的日均频次分布。
* **计算波动率：** 每天计算该词频次的平均值（$\mu$）和标准差（$\sigma$）。然后计算变异系数 $CV = \sigma / \mu$。
* **自学习逻辑：** * 真正的热点词（如突发事件），平时 $\mu$ 很低，突然爆发会导致 $\sigma$ 极大，$CV$ 值远大于 1。
* “群组”、“频道”这种广告词，每天发帖量极其稳定，$CV$ 值会趋近于 0（极其平滑）。
* **规则闭环：** 当系统发现某个词连续 N 天处于极高频，且 $CV < 0.2$ 时，**系统自动将其加入“动态停用词库（Dynamic Stopwords）”**，实现无需人工干预的自适应过滤。



### 2. 共现词“毒性传染”模型（自动扩充黑名单）

**自学习目标：** 你人工提供几个“种子黑名单”（如“赌场”、“发帖”、“收款”），系统通过学习，自动把新的黑话和变种广告词找出来。
**算法实现：简易马尔可夫共现矩阵**

* **构建关联：** 每次提取分词后，统计同在一个句子中出现的词对（如 `{"广告", "招商"}: 50次`）。
* **计算毒性传递：** 设定一个基础毒性池（包含你的硬编码黑名单）。
* **自学习逻辑（PageRank的极简版）：** * 如果一个未知词（比如“上押”）在历史统计中，有 80% 的概率是和“收款”、“频道”这些高毒性词汇**同时出现**的。
* 系统就会把“收款”的毒性分数传染给“上押”。
* 当“上押”的毒性分数超过阈值，系统**自动将其标记为垃圾词**并从热词榜单中降权。这就是基于上下文的无监督学习。



### 3. 信誉积分衰减引擎（基于来源画像）

**自学习目标：** 学习“谁”在说话，而不是只看“说”了什么。垃圾词通常由特定的机器号或固定的引流频道发出。
**算法实现：TF-ITF (Term Frequency - Inverse Trust Frequency)**

* **用户/频道画像：** 结合你系统中的**去重引擎（Dedup Engine）**。如果某个用户或频道经常发送高度重复的消息，系统对其“信誉积分（Trust Score）”进行扣分。
* **自学习逻辑：** * 在提取热词时，不再是 `1次出现 = 1分`。
* 而是 `得分 = 1次出现 × 该消息发送者的信誉积分`。
* 如果“群组”这个词出现了 500 次，但其中 450 次来自于信誉分为 0.1 的引流机器人，那么它的有效热度只有 45 + 50 = 95 分。
* 如果“国漫”出现了 100 次，来自于信誉分为 1.0 的正常活跃群友，它的有效热度就是 100 分。“国漫”自动反超“群组”。



### 4. 消息结构指纹特征提取（形态学过滤）

**自学习目标：** 让系统学习“广告消息长什么样”，从而间接降低这整条消息中所有词的权重。
**算法实现：启发式特征打分**
广告通常具有强烈的形态学特征，不需要理解语义即可识别。
在自学习模块中为每条消息打一个 `Spam_Shape_Score`（结构垃圾分）：

1. **链接密度：** 文本字数 / 包含的 URL 数量。低于阈值加垃圾分。
2. **特殊符号密度：** 统计 `👉, 👇, 🔥, 🚀, 💰, ✈️, ☎️` 等表情符号的占比。
3. **换行/排版规律：** 广告喜欢用大量的列表和空行。

* **自学习逻辑：** * 系统记录每个词所属消息的平均 `Spam_Shape_Score`。
* 如果“排行榜”这个词，90% 都出现在垃圾形态分极高的消息中，系统就会“学习”到这是一个广告常用词，自动调低其热点提取权重。



### 总结：如何重构你的自学习代码流

在你的 `tg-one` 项目中，不加任何 AI，只需在统计算法中加上时间维度和关联维度，即可实现极高的准确度。

**建议的代码数据流结构：**

1. `Message` 进场 -> `形态学打分` (判断是否是广告格式)。
2. 提取分词 -> 记录 `(Word, 频道ID, 用户ID, 形态学分数)` 到 SQLite/DuckDB 缓存表。
3. 夜间/定时 Summary 任务启动：
* 运行 **变异系数计算**，剔除 $CV$ 极低的词（杀掉“群组”、“频道”）。
* 运行 **信誉度计算**，降权来自刷屏机器人的词。
* 运行 **毒性传染网络**，更新动态黑名单。


4. 输出清洗后的“真·热点词”。

这种纯统计的自学习方法，计算开销极小（普通的 SQL 查询和基本的 Python Math 操作即可完成），且运行时间越长，基线越准确，过滤效果越好。

我们继续深挖。既然你的目标是**“纯算法、无外部依赖、极致的自动化与准确度”**，并且我注意到你的代码库 `core/algorithms/` 目录下其实已经储备了非常多优秀的底层算法工具（如 `hll.py`, `simhash.py`, `lsh_forest.py`, `ac_automaton.py`, `bloom_filter.py`），我们可以直接将这些现成的“核武器”组装进热词系统中，进行一次**降维打击**。

以下是针对热词提取与自学习算法的**深度迭代方案（第 3 版）**：

### 一、 整合现有底层算法（盘活你的 `core/algorithms`）

你不需要从头造轮子，可以直接利用代码库中已有的算法模块来重构热词的过滤与统计管道：

#### 1. 使用 HLL (HyperLogLog) 实现极低内存的“用户/群组去重”

前面提到的 TF-IUF（逆用户频率）需要统计“有多少个不同的人/群提到了这个词”。如果用传统的 `Set` 存 ID，内存会爆炸。

* **迭代方案**：利用你写好的 `core/algorithms/hll.py`。
* **落地**：为每个候选热词维护两个 HLL 实例：`user_hll` 和 `chat_hll`。
每次出现分词，直接 `word.user_hll.add(user_id)`。
结算时，$IUF权重 = \log(总活跃用户数 / word.user_hll.count())$。**耗时几乎为 0，内存占用仅需几 KB。**

#### 2. 使用 SimHash + LSH Forest 拦截“变种广告” (前置阻断)

广告发布者会故意在“发帖”、“收款”中间加空格、特殊符号来绕过统计。

* **迭代方案**：在消息进入热词分词器**之前**，利用 `simhash.py` 和 `lsh_forest.py` 计算整条消息的局部敏感哈希。
* **自学习联动**：如果某条消息因为触发了高毒性词（见上一版的共现模型）被判定为广告，将其 SimHash 存入 LSH Forest。后续进来的消息如果和它高度相似（哪怕改了几个字），**直接跳过分词阶段**，根本不给广告词进入候选池的机会。

#### 3. 使用 AC 自动机实现 $O(N)$ 的动态黑名单秒杀

* **迭代方案**：自学习算法每天产出的“动态停用词（Dynamic Stopwords）”，直接喂给 `ac_automaton.py` 构建状态机。
* **落地**：分词引擎处理前，文本先过一遍 AC 自动机。只要匹配到动态黑名单里的词组，直接将其前后的上下文降权。这比传统正则匹配快几个数量级。

### 二、 统计算法再升级：引入“基尼系数”与“牛顿冷却定律”

如果变异系数（CV）和 Z-Score 还不够，我们可以引入社会学和物理学的统计算法。

#### 1. 空间分布特征：基尼系数 (Gini Coefficient)

基尼系数用来衡量财富的不平等，在这里我们用来**衡量词汇分布的不平等**。

* **现象**：“频道”、“群组”等广告词在所有监控的群组里，出现的次数非常均匀（极度平均）。而真正的热点（如“国漫”、“财报”）通常只在特定的几个群里爆发讨论（极度不平均）。
* **算法**：
将一个词在各个群组出现的次数排成数组：$[c_1, c_2, c_3 ... c_n]$。
计算其基尼系数（0 到 1 之间）。
* **基尼系数趋近 0**（大家都在发）：平台基建词或全网牛皮癣，**直接降权封杀**。
* **基尼系数趋近 1**（少数群在疯狂刷）：局部突发热点，**加权置顶**。



#### 2. 时间衰减特征：牛顿冷却算法 (Newton's Law of Cooling)

目前你的统计可能是基于“自然日”（每天一个榜单），这种硬切断会导致跨夜的热点断层，也会让平稳积累的广告词有机会在深夜登顶。

* **迭代方案**：给每个词的热度加上“时间衰减因子”。公式为：
$当前热度 = 历史热度 \times e^{-\lambda \Delta t} + 本次新增频次 \times 权重$
* **效果**：真正的热点词，新增频次极大，热度瞬间飙升；一旦没人讨论了，热度会随时间指数级“冷却”下去。而广告词因为有 $IUF$ 和基尼系数的压制，每次新增的分数极低，永远只能保持在“温水”状态，无法沸腾。

### 三、 无监督自学习的高阶玩法：句法模板提取 (Template Induction)

除了学习“单个词”是否有毒，系统还能不能自己学习“广告的说话方式”？完全可以，基于**序列模式挖掘**。

#### 1. 高频子序列提取 (Frequent Substring Mining)

抛弃传统的依靠字典的分词，对所有含有“群组”、“收款”这类基石垃圾词的消息，进行 N-Gram 子序列提取。

* **算法过程**：
系统自动发现 `["高清", "无码", "点击", "进群"]` 这个序列在过去 3 天内高频共现。
* **抽象化**：
系统自动将具体数字和标点替换为通配符，生成一个**“广告句法模板”**：`*高清*点击*链接*`。

#### 2. 贝叶斯惊喜度 (Bayesian Surprise) 检测热点

人们在讨论真实热点时，使用的词汇组合往往是**出乎意料**的。而广告是**陈词滥调**。

* **算法**：
维护一个简单的二元词序列概率矩阵（比如 `(群组 -> 链接)` 的概率是 0.8）。
当遇到一句话时，计算这句话的“联合概率”。
* **如果概率极高**：这是套话/广告，从热词池剔除。
* **如果概率极低（惊喜度极高）**：说明出现了新的话题组合（比如 `(苹果 -> 暴跌)`），系统立刻捕捉这些词作为潜在的**超级热点**。

### 🚀 终极架构落地建议 (MVP Pipeline)

你可以用极少的代码，将以上逻辑串联成一条过滤流：

1. **入口过滤 (LSH / SimHash)**：新消息进来，计算局部哈希，落入广告簇的，直接丢弃。
2. **文本清洗 (AC Automaton)**：利用动态生成的黑名单词库，秒杀文本中的诱导词汇。
3. **特征统计 (HLL)**：用 HLL 以 $O(1)$ 空间更新该词的 `Unique_Users` 和 `Unique_Chats`。
4. **积分打分 (Math)**：

$$FinalScore = TF \times \log(\frac{Total}{UserHLL}) \times Gini(ChatDistribution) \times DecayFactor$$


5. **夜间闭环 (Self-Learning)**：定时任务扫出 $Gini < 0.2$（极其均匀）且波动率极低的词，自动追加到 AC Automaton 黑名单中。

通过这条 Pipeline，系统不需要理解任何中文语义，也不用调用任何耗时的 AI 接口，仅仅依靠**空间分布不均（Gini）**、**受众集中度（HLL）**和**历史衰减对比**，就能让 Telegram 上的垃圾广告彻底在榜单中隐形。
